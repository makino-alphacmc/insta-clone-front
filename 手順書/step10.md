# Step 10: リリース 🎉

## 📋 このステップでやること

アプリケーションをリリースし、次のフェーズの計画を立てます。

## ✅ リリース手順

### 10-1) Git タグの作成
※ 目的: リリース時点のスナップショットを固定し、再現性とロールバックを確保。

プロジェクトにGitタグを打ちます：

```bash
# プロジェクトのルートディレクトリで
cd ~/work

# タグを作成
git tag -a v0.1.0 -m "Initial release: Instagram Clone MVP"

# タグをプッシュ（リモートリポジトリがある場合）
git push origin v0.1.0
```

### 10-2) 本番環境へのデプロイ
※ 目的: 実際の利用環境へ反映し、ユーザーがアクセスできる状態にする。

本番環境にデプロイします：

```bash
# サーバーに接続
ssh user@your-server-ip

# デプロイスクリプトを実行
./deploy.sh
```

### 10-3) 動作確認
※ 目的: 本番でフロント/API/画像表示が意図通りつながるか最終確認。

本番環境で以下を確認します：

- [ ] フロントエンドが表示される
- [ ] APIが動作する（`/health` エンドポイント）
- [ ] 投稿機能が動作する
- [ ] 画像がアップロードされ、表示される
- [ ] タイムラインに投稿が表示される

### 10-4) URL の共有
※ 目的: 公開先をチームやユーザーに共有し、検証や利用を開始できるようにする。

アプリケーションのURLを共有します：

- **フロントエンド**: `http://your-server-ip:3000` または `https://your-domain.com`
- **API**: `http://your-server-ip:8000` または `https://api.your-domain.com`

## 🎊 リリース完了！
※ 意味: MVPとしてのゴールに到達した区切り。ここから先は改善フェーズ。

おめでとうございます！Instagram クローンの MVP（最小限の機能）がリリースされました。

## 🐛 トラブルシューティング

問題が発生した場合は、以下を確認してください：

1. **ログの確認**:
   ```bash
   docker compose logs -f
   ```

2. **コンテナの状態確認**:
   ```bash
   docker compose ps
   ```

3. **環境変数の確認**:
   - `.env` ファイルが正しく設定されているか
   - Supabase のキーが正しいか

4. **ネットワークの確認**:
   - ポートが正しく開いているか
   - ファイアウォールの設定
